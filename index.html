Crabbox

Generic remote execution control plane. Agents get warm, isolated, fully-audited "hands" anywhere.

Always-ready agent worker boxes (see pool ensure, prewarm)

THE AGENTIC LOOP

Observe → Manipulate → Execute → Reflect

A tight, auditable cycle. The agent owns the intelligence. Crabbox owns safe execution of every step that touches data or systems.

01
OBSERVE
Agent inspects Airbyte connection states, samples live data from sources/destinations/warehouse, reviews previous Crabbox run artifacts, history, and metrics.
02
MANIPULATE
AI decides: update Airbyte connection? Generate custom CDK source/transform? Trigger reverse activation? Plans exact Crabbox commands + code.
03
EXECUTE
All sensitive or custom work runs through Crabbox on warm pooled boxes: crabbox run --pool with scripts, shell, artifacts, and strict env allowlisting.
04
REFLECT
Agent ingests timing, artifacts (CSVs, reports), JUnit metrics, phases, and Airbyte state. Updates world model. Loops or declares goal complete.

1. Capacity & Preparation
(once per pool / on ref change)
crabbox pool ensure example-org/data-pipelines/main/hetzner/linux/cpx51 --min-ready 2 --create -- --provider hetzner --type cpx51 --cache-volume airbyte-pip
Registers warm leases. Uses prewarm + actions-hydration (from your .github/workflows/airbyte-stack.yml) to install Airbyte CLI + CDK + Python deps exactly like your CI. See docs/commands/pool.md and actions-hydration.md.
2. Agent Observes
Agent (via tools or framework) queries warehouse state, lists Airbyte connections, reviews previous crabbox history --label airbyte-* runs, downloads prior artifacts.
3. Agent Manipulates (AI brain)
Decides: "Standard forward via Airbyte" or "Custom source + heavy transform" or "Reverse activation with personalization". Generates Airbyte config or Python script using Airbyte CDK / PyAirbyte. Plans the exact crabbox run invocation.
4. Execute (via Crabbox — the hands)
crabbox run --pool example-org/data-pipelines/main/hetzner/linux/cpx51 \ --script /tmp/agent_airbyte_sync.py \ --allow-env 'AIRBYTE_API_KEY,WAREHOUSE_*,DEST_*' \ --env-from-profile ~/.agent/creds.env \ --artifact-glob 'reports/**,outputs/*.csv' \ --junit airbyte-run.xml \ --label "agent:airbyte-src-dst:$(date -I)"
Runs on pre-hydrated box. Secrets forwarded safely (never in code or prompts). Full artifacts, results, phases, and history captured. See env-forwarding.md and artifacts.md.
5. Reflect + Iterate
Agent reads artifacts (data samples, error CSVs), JUnit metrics, timing, and Airbyte sync status. Updates internal state. Triggers next manipulation or reverse step. Every iteration is a first-class audited record.
Key Crabbox guarantees used here (from the docs):
• Warm reusable workers via pools + prewarm (no repeated cold boots or full dep installs)
• Repo-owned setup via actions-hydration (your .github/workflows installs Airbyte + tools — parity with CI)
• Strict secret discipline via env allowlist + profiles (critical for agent use of real creds)
• Complete observability: every agent action produces artifacts, results, phases, and history
• Works for reverse / on-prem / custom via SSH providers, Tailscale ponds, delegated sandboxes
IN PRACTICE

Real agent invocation (what actually runs)

# Agent decides forward sync + custom reverse activation is needed
# 1. Ensure / borrow warm box (with Airbyte + CDK pre-installed via hydration)
crabbox pool ensure example-org/data-pipelines/main/hetzner/linux/cpx51 --min-ready 1

# 2. Agent-generated: configure standard Airbyte forward (audited)
crabbox run --pool example-org/data-pipelines/main/hetzner/linux/cpx51 \
  --shell 'airbyte connections create \
    --source-id new-billing-events \
    --destination-id warehouse-snowflake \
    --schedule 15min' \
  --allow-env 'AIRBYTE_*' \
  --label "agent:configure-forward:$(date -I)"

# 3. Agent-generated custom reverse script (AI enrichment + eligibility)
cat > /tmp/agent_reverse.py << 'PY'
# ... uses snowflake + salesforce SDK + custom AI logic ...
print("CRABBOX_PHASE:reverse-activation rows=1247 updated=312")
PY

crabbox run --pool example-org/data-pipelines/main/hetzner/linux/cpx51 \
  --script /tmp/agent_reverse.py \
  --allow-env 'WAREHOUSE_SNOWFLAKE_*,DEST_SALESFORCE_*,AI_*' \
  --env-from-profile ~/.agent/data-creds.env \
  --artifact-glob 'reports/reverse-*.csv' \
  --junit reverse-activation.xml \
  --label "agent:reverse-activation:$(date -I)"

# 4. Agent reflects on artifacts + results + Airbyte status, loops or done

Pre-warm the pool once (or use pool ensure in your agent orchestrator). Subsequent runs are seconds. Every line the agent "writes" or executes is versioned in git and fully recorded in Crabbox history.

Every Airbyte config change, every custom CDK script, every reverse activation is a first-class crabbox run with artifacts, phases, JUnit, and immutable history. Perfect for compliance and agent self-reflection.
Safe by construction

Fast, cheap iteration
Warm pools + cache volumes + actions-hydration mean agents get instant execution environments. No 5-minute cold starts. Pre-hydrate the exact Airbyte + data stack your agents need.
