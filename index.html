A one-page tutorial for the vision

Tell the harness the outcome. It borrows workers and brings back proof.

Agentic Airbyte is not "an agent poking random tools." It is a clean operating model: the harness thinks, Crabbox sandboxes execute, and Airbyte plus custom code move the data with evidence every step of the way.

The promise is simple.

Users should describe what data outcome they want. The system should decide what work is needed, run that work in safe remote boxes, and report exactly what happened.

1
Intent, not pipeline babysitting.Ask for freshness, activation, enrichment, repair, or validation.

Execution, not hand-waving.Every movement happens inside a real sandbox worker.
3
Proof, not trust me.Artifacts, phases, metrics, results, and history return to the harness.
The story

One goal becomes one governed worker episode.

This is the end-to-end narrative the website needs to teach. It is intentionally small: five beats, one loop, no giant architecture wall.

1. A goal arrives.

"Keep customer profiles enriched from the warehouse. Refresh daily. Respect the data policy."

  • The caller says the outcome, not every shell step.
  • The harness records the policy, deadline, and success criteria.

2. The harness decides what kind of work this is.

It checks state, prior Crabbox history, schemas, destination rules, and the last proof bundle.

  • Standard pipe? Use Airbyte inside an ETL worker.
  • Activation, scoring, enrichment, or edge network? Pick a custom worker pool.

3. Crabbox leases the right sandbox worker.

The harness asks Crabbox for capacity. Crabbox provides an isolated, repo-hydrated worker from a ready pool.

  • Warm pools make dispatch fast.
  • Profiles and allowlists keep secrets scoped to the task.

4. The worker moves the data and writes proof.

Airbyte handles the reliable source-to-destination work. Custom code handles scoring, activation, validation, and policy-specific steps.

  • The worker emits phases, tests, row counts, and artifacts.
  • The harness never has to hold raw data or long-lived credentials.
Tutorial loop

Every run teaches the next run.

The important idea is not just "run a connector." It is a closed loop: observe, plan, lease, execute, ingest proof, and decide again.

Observe
Look at warehouse state, destination freshness, prior run history, and the current policy.
Plan
Choose ETL, reverse ETL, enrichment, validation, or repair. Pick the matching worker pool.
Lease
Use Crabbox to borrow a warm sandbox with the right repo, runtime, cache, and secret scope.
Execute
Run Airbyte, CDK, Python, dbt, model calls, or custom activation logic in the worker.
Reflect
Read artifacts and results. Retry, alert, mark complete, or write a better plan next time.

What this is really selling

Not another static pipeline UI. Not a loose agent with raw credentials. The product promise is agentic intent with governed execution.

The harness can be LangGraph, a scheduler, or normal service code. The execution contract is still simple: crabbox run --pool ...

Why it feels trustworthy

The magic is not hidden. It is returned as evidence.

For agents to own data movement, they need memory they can trust. Crabbox gives the harness structured run output instead of vague logs.

History
Which worker ran, when it ran, what command was used, and what happened.
Phases
Named execution moments through CRABBOX_PHASE, so the harness can understand progress and failure.
Results
JUnit, validation checks, row counts, timings, retry details, and task status.
Artifacts
Reports, generated configs, samples, connector output, policy proof, and replay bundles.
minimal-harness.sh 
# 1. Keep ETL workers warm.
crabbox pool ensure example-org/data-movement/main/provider/linux/etl \
  --min-ready 3 --create -- \
  --cache-volume airbyte-etl

# 2. Run Airbyte inside a borrowed sandbox.
crabbox run --pool example-org/data-movement/main/provider/linux/etl \
  --shell 'python -m workers.airbyte_ingest --config /tmp/generated.json' \
  --allow-env 'AIRBYTE_*,SOURCE_*,WAREHOUSE_*' \
  --env-from-profile etl-warehouse \
  --artifact-glob 'reports/**,metrics.json' \
  --junit reports/

# 3. Feed proof back to the harness.
crabbox history --limit 50 --json
crabbox results <run-id> --json
crabbox artifacts download <run-id>
