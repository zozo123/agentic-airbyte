The Core Idea

A new version of ETL & rETL

Traditional ETL and reverse ETL are static, human-maintained pipelines. They are brittle, slow to change, and offer almost no room for AI agents to participate intelligently.

The new notion is simple and powerful:

AI agents intelligently drive data movement by using Crabbox remote sandboxes — clean, warm, isolated execution environments — where they can safely run Airbyte for standard movement and custom logic for everything complex, reverse, or dynamic.

How It Actually Works

The End-to-End Agentic Flow

A tight, observable loop. The agent reasons, uses Crabbox to run work inside real remote sandboxes, gets structured results back, and iterates.

flowchart TD subgraph Loop ["Agentic Loop"] A[Agent observes state
warehouse + Airbyte + past runs] A --> B[Agent decides ETL or rETL + generates logic] B --> C[Crabbox: Get or borrow remote sandbox
pool ensure + prewarm] C --> D[Sandbox ready + hydrated with Airbyte + tools] D --> E1[ETL Path] E1 --> F1[Run Airbyte or custom extract/load inside sandbox] D --> E2[rETL Path] E2 --> F2[Read warehouse → AI logic → push to destinations] F1 & F2 --> G[Sandbox returns artifacts + results + history] G --> A end
ETL (Forward)
Agent runs data into the warehouse
  1. 1. Leases a clean remote sandbox via Crabbox (warm pooled or on-demand).
  2. 2. Inside the sandbox: runs Airbyte to pull from sources (DBs, SaaS, APIs).
  3. 3. Optionally runs AI-generated transforms or custom logic in the same isolated environment.
  4. 4. Loads to warehouse/destinations. Returns full artifacts and metrics to the agent.
rETL (Reverse)
Agent activates data out to systems
  1. 1. Same remote sandbox (reused from pool for speed).
  2. 2. Reads modeled data from warehouse.
  3. 3. Applies AI-driven logic (scoring, personalization, eligibility, enrichment).
  4. 4. Pushes to operational systems (Salesforce, Slack, internal apps, on-prem). Full audit returned.

Every step runs inside a Crabbox-provided remote sandbox. The agent gets real evidence back to reason over. No ad-hoc scripts. Full reproducibility and audit.

The Two Layers

The Tools

Airbyte

THE RELIABLE PIPES

Open-source data integration platform with 300+ connectors and excellent observability.

  • • Standard src → dst movement (to warehouse or direct)
  • • CDK for custom connectors when needed
  • • CLI, API, and Python libraries agents can drive programmatically

Crabbox

THE REMOTE SANDBOX LAYER

Gives agents clean, warm, isolated remote execution environments with full safety and audit.

  • • Warm pooled sandboxes (seconds, not minutes)
  • • Repo-native setup via Actions hydration (Airbyte + your tools)
  • • Strict secret forwarding + per-lease isolation
  • • Artifacts, results, and complete history for the agent to observe
Getting Started

How to Use This

1
Prepare your data-ops repo
Add crabbox.yaml. Point actions.workflow to a GitHub Actions file that installs Airbyte + Python + CDK. Use cache volumes for speed.
2
Ensure remote sandbox capacity
crabbox pool ensure my-pipelines/main/hetzner/linux/cpx51 --min-ready 2 --create -- --provider hetzner --cache-volume airbyte-etl
Warm, reusable sandboxes with your exact stack ready for agents.
3
Give your agent Crabbox as a tool
The agent can now call crabbox run --pool ..., generate scripts, and execute Airbyte or custom logic inside real remote sandboxes.
4
Let the agent drive
Example the agent will produce:
crabbox run --pool ... --shell 'airbyte ...' --allow-env 'AIRBYTE_*' --artifact-glob 'reports/**'
or a generated Python script for complex rETL.
Every action leaves artifacts, metrics, and history. The agent observes real results and iterates. All powered by real Crabbox features (pools, prewarm, actions hydration, env forwarding, artifacts) and Airbyte connectors.
